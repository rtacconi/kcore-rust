# Licensing and editions

kcore uses an **open-core** model.

**Community Edition** is the **Apache License 2.0** source tree. The core stays open and buildable from source. **Apache-2.0 does not restrict commercial use** of that code; what you build and run under that licence is governed by Apache-2.0, not by a subscription.

**Subscriptions** are for **services and artifacts around the project**: the **official supported distribution**, **stable production update channel**, **official signed ISO images**, **tested updates**, and **commercial support**. A subscription does **not** replace or revoke the open-source licence on the public core. It is not a substitute for reading `LICENSE` and the Apache-2.0 terms.

**Recommended usage**

| Context | Edition |
|--------|---------|
| Contributors, labs, homelabs, evaluation | Community Edition (build from source) |
| Production | kcore Standard or kcore Premium (subscribed nodes) |

For prices, support tiers, and socket-based billing, see [PRICING.md](../PRICING.md) in the repository root.

For the **partner program** (resellers, integrators, service providers), email [team@tacconiconsulting.com](mailto:team@tacconiconsulting.com?subject=kcore%20partner%20program).
